🎯 Conceito Central
O jogador controla um "pacote de dados" que precisa viajar pela hierarquia de memória para chegar ao processador. Cada zona do mapa é uma camada de memória diferente.

🗺️ O Mapa — Zonas de Memória
[ HD / SSD ] → [ RAM ] → [ Cache L3 ] → [ Cache L2 ] → [ Cache L1 ] → [ CPU ]
   Lento 🐢                                                              Rápido ⚡
Cada zona tem:

Velocidade diferente do personagem (na RAM anda devagar, na Cache voa)
Visual único (cores, texturas, obstáculos diferentes)
Inimigos/obstáculos temáticos (fragmentação, page fault, miss)


🕹️ Controles
DispositivoControle📱 CelularBotões de seta na tela💻 PCTeclas WASD ou setas

⚔️ Obstáculos por Zona

HD: Fragmentação de disco — caminhos bloqueados aleatoriamente
RAM: Page faults — buracos que reenviam ao início da zona
Cache L3/L2: Cache misses — inimigos que te empurram de volta
Cache L1: Guardas de proteção — puzzles rápidos para passar


🏆 Sistema de Pontuação

⏱️ Quanto mais rápido, mais pontos
💥 Cada obstáculo atingido = penalidade de tempo
⭐ Coletar "bits" no caminho = bônus
📊 Ranking salvo localmente


📚 Elemento Educativo
Ao entrar em cada zona aparece um card rápido:

"Cache L1 — Memória mais rápida, mais próxima da CPU. Latência: ~1ns"


🎨 Visual

Estilo neon/tech (fundo escuro, elementos brilhantes)
Personagem: um pacote de dados pulsando
Cada zona com cor própria: HD=vermelho, RAM=laranja, Cache=azul, CPU=verde
